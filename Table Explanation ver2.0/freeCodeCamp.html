Introduction

Imagine a spreadsheet - rows and columns filled with information. That's precisely what the <table> element allows you to create in HTML. It establishes a container for organizing your data into a grid layout, making it visually clear and easy to navigate for users.

Description

The <table> element by itself doesn't hold the actual data. It serves as a framework for other elements that define the structure:

  • Rows (<tr>): These elements represent horizontal lines within the table. Each <tr> element represents a single row of data.

  • Cells (<td> and <th>): These elements define the individual compartments within a table. They hold the actual content you want to display. There are two main cell types:

    • Data Cells (<td>): Used for the main content of the table. They typically display information related to the corresponding row header.

    • Header Cells (<th>): Used for labeling rows or columns. They are often bolded and visually distinct from regular data cells to improve readability.

Example

Here's a more comprehensive example of a table with headers, data cells, and proper formatting:

Code:

<table border="1">
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>City</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Alice</td>
            <td>30</td>
            <td>New York</td>
        </tr>
        <tr>
            <td>Bob</td>
            <td>25</td>
            <td>London</td>
        </tr>
    </tbody>
</table>

Result:

Name Age City
Alice 30 New York
Bob 25 London
Uses

Tables are incredibly versatile and can be used for various functionalities on web pages:

  • Data Presentation: The primary use case is presenting data sets in a structured and organized manner. This is ideal for financial records, product listings, comparison charts, or any information that benefits from a grid layout.
  • Layout Purposes: Tables can be used for basic layout structures, especially in older web design practices. You can create sections within a page using tables, although this approach is less common with the emergence of more advanced CSS techniques.
Commonly Used Attributes

While the <table> element itself has a few attributes, most customization happens with the child elements (<tr> and <td>). Here are some key attributes to remember:

  • border (deprecated): Defines the width of the table's border in pixels. It's recommended to use CSS for border styling as it offers more flexibility.
  • cellpadding: Defines the space in pixels between the cell content and its border. This creates padding around the data within the cell.
  • cellspacing: Defines the space in pixels between adjacent cells. This controls the spacing between individual cells in the table.
  • summary: Provides a brief description of the table's purpose. This is helpful for accessibility, especially for screen readers used by visually impaired users.
